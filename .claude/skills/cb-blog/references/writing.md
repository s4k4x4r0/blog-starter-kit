# 記事執筆フェーズ

## やること

1. 以下を読んで、記事の材料を集める:
   - `NOTES.md`（検証の経緯、ブログネタ）
   - `poc/` のコード（実装の詳細）
   - `project.yaml`（記事のメタ情報）
2. `blog/<article-id>/draft.md` に記事を書く

## 記事のスタイル

### 構成

```markdown
# タイトル

導入（何を・なぜ・読者が得られるもの）

## 前提・環境

## 手順1: ...
## 手順2: ...
...

## まとめ
```

### 書き方のルール

- 技術記事として正確かつ実用的に書く
- 実際に試した結果に基づいて書く（推測で書かない）
- コードブロックには言語を指定する
- 長すぎるコードは要点を抜粋し、全体はGitHubリポジトリへのリンクで補足する
- ハマりポイントや注意点は明示する（NOTES.md の `-> ブログ:` マーカーを参考に）
- 画像を使う場合は `blog/<article-id>/images/` に配置する
  - Markdown内では相対パスで参照する: `![説明](images/xxx.png)`

### トーン

- 「〜してみた」「〜を試す」のような実験・体験ベースのトーン
- 堅すぎず、カジュアルすぎず

## phase の更新

下書きが完成したら `phase: draft` -> `phase: review` に更新してください。
