# 検証計画フェーズ

## やること

1. `project.yaml` と `NOTES.md` を確認し、記事の方向性を把握する
2. 検証の計画を立てる:
   - 何を検証するか（ゴール）
   - どう検証するか（手順）
   - 使用する技術スタック
   - 想定される成果物
3. `poc/` ディレクトリにプロジェクトの雛形を作成する
4. 計画を `NOTES.md` に記録する

## poc/ ディレクトリの構成

検証コードは `poc/` 直下に配置してください。
言語やフレームワークに応じた標準的なプロジェクト構成に従ってください。

例（Kotlin + Gradle の場合）:
```
poc/
├── build.gradle.kts
├── settings.gradle.kts
└── src/
    └── main/
        └── kotlin/
```

## NOTES.md への記録

検証計画の概要を `## 方針` セクションに記録してください。

## phase の更新

計画がまとまり検証を開始する段階で `phase` は `poc` のままにしてください。
検証が進行中であることを示します。
