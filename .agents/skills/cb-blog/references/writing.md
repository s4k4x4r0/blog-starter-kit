# 記事執筆フェーズ

## やること

1. 以下を読んで、記事の材料を集める:
   - `NOTES.md`（検証の経緯、ブログネタ）
   - `poc/` のコード（実装の詳細）
   - `project.yaml`（記事のメタ情報）
2. `blog/<article-id>/draft.md` のテンプレートに沿って記事を書く

テンプレート: [templates/draft.md](../templates/draft.md)
テンプレートはユーザーが自由にカスタマイズしている可能性があります。テンプレートの構成に従ってください。
テンプレート内の HTML コメント（`<!-- ... -->`）にはユーザーからの指示が書かれています。各セクションに何を書くべきかの指針になるので、コメントの内容に従って執筆してください。

## 書き方のルール

### コードブロック

- 言語指定を必ず付ける（```typescript, ```yaml 等）
- コードブロックの前に、何を見せるかを1文で説明する
- コードブロックの後に、ポイントや注意点を箇条書きで説明する
- 長すぎるコードは要点を抜粋し、全体はGitHubリポジトリへのリンクで補足する

### 画像

- 画像は `blog/<article-id>/images/` に配置する
- Markdown内では相対パスで参照する: `![説明](images/xxx.png)`
- WordPress MCP が自動でアップロード・URL置換する
- 命名規則は `AGENTS.md` の「スクリーンショット画像」セクションを参照

#### POC の画像を記事に使う場合

`poc/images/` から `blog/<article-id>/images/` にコピーしてください。
`NOTES.md` で `-> ブログ:` マーカーが付いた画像を確認し、記事に必要なものを選んでください。

#### 記事用スクリーンショットの撮影

記事用のスクリーンショットは直接 `blog/<article-id>/images/` に保存してください:

```bash
agent-browser open "http://localhost:3000"
agent-browser screenshot blog/<article-id>/images/result-screen.png
```

### リンク

- 公式ドキュメントや参考記事は積極的にリンクする
- GitHubリポジトリの URL は目立つように単独行で記載する

### その他

- 実際に試した結果に基づいて書く（推測で書かない）
- ハマりポイントや注意点は明示する（NOTES.md の `-> ブログ:` マーカーを参考に）

## WordPress MCP で使えない記法

以下の Markdown 記法は WordPress MCP で正しく変換されないため、使わないでください:

- **コードブロックのファイル名指定**: ````typescript:app.ts` のような記法は効かない。ファイル名はコードブロックの直前にテキストで書く
- **タスクリスト**: `- [ ]` / `- [x]` は使わない。通常のリストで代替する
- **H1 はタイトル専用**: H1 は投稿タイトルに自動変換され本文から除去される。本文の見出しは H2 以下を使う

## phase の更新

下書きが完成したら `phase` は `draft` のままにしてください。WordPress投稿後に `posted` に変わります。
