# 機密情報のマスク処理

スクリーンショットに含まれる機密情報（AWS アカウント ID、メールアドレス等）を矩形で塗りつぶす。

## 手順

### 1. 座標の特定

Read ツールでスクリーンショットを表示し、機密情報の位置を目視で確認する。
画像のサイズから大まかな座標（左上 x1,y1 と右下 x2,y2）を推定する。

座標の推定が難しい場合は、余裕を持って広めにマスクすること（機密情報が見切れるより、少し広めに隠す方が安全）。

### 2. マスク処理の実行

```bash
uv run .agents/skills/screenshot/scripts/mask-image.py <画像パス> "x1,y1,x2,y2" ["x1,y1,x2,y2" ...]
```

例（アカウント ID とメールアドレスをマスク）:
```bash
uv run .agents/skills/screenshot/scripts/mask-image.py \
  poc/images/aws-lambda-function-list.png \
  "980,5,1130,20" \
  "950,30,1270,45"
```

オリジナル画像は `/tmp/backup/` に自動バックアップされる。

### オプション

| オプション | デフォルト | 説明 |
|---|---|---|
| `--color R,G,B` | `0,0,0`（黒） | マスクの色を変更する |

### 3. 結果の確認

マスク後の画像を Read ツールで確認し、以下をチェックする:
- 機密情報が完全に隠れているか
- マスク範囲が広すぎて必要な情報まで隠れていないか

座標がずれていたら調整して再実行する（オリジナルは `/tmp/backup/` に退避済み）。

## チェックすべき機密情報

| 種類 | 画面上の典型的な表示位置 |
|---|---|
| AWS アカウント ID（12桁） | ヘッダー右上、リソース ARN 内 |
| メールアドレス | ヘッダー右上のユーザー名 |
| IAM ユーザー名 / ロール名 | ヘッダー、ブレッドクラム |
| API キー / シークレット | 設定画面、環境変数一覧 |
| IP アドレス | セキュリティグループ、ログ |
| S3 バケット名（推測可能なもの） | リソース一覧 |

## 注意事項

- 座標は画像の解像度に依存する。撮影条件が変わると位置がずれるので注意
- マスク漏れがないか、最終確認は必ず Read ツールで行う
