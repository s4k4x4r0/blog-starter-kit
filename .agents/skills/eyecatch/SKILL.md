# eyecatch: アイキャッチ画像生成

ベース画像にタイトルテキストを重ねてアイキャッチ画像を作成する。

## 生成の流れ

1. タイトルテキストを決める（`$ARGUMENTS` にあればそれを使う）
2. 出力先を決める
   - 記事ディレクトリがあれば `blog/<article-id>/images/eyecatch.png`
   - なければプロジェクトルートに `eyecatch_output.png`
3. スクリプトを実行する
4. **生成された画像を Read ツールで確認する**
5. 問題があればパラメータを調整して再生成する

## スクリプトの実行

プロジェクトルートから実行する:

```bash
uv run .agents/skills/eyecatch/scripts/make_eyecatch.py \
  .agents/skills/eyecatch/assets/eyecatch.base.png \
  "タイトルテキスト" \
  出力先パス \
  [--size フォントサイズ]
```

`--size` のデフォルトは 64px。

### 改行位置の指定

タイトル中に `\n` を入れると、その位置で強制改行できる。
スクリプトにもカタカナ語保護や行均等化の自動処理があるが、意味的に自然な改行位置はLLMの方が正確に判断できるため、長いタイトルでは `\n` を積極的に使うこと。

```bash
# シェルで \n を渡すには $'...' 記法を使う
uv run .agents/skills/eyecatch/scripts/make_eyecatch.py \
  .agents/skills/eyecatch/assets/eyecatch.base.png \
  $'GitHub Actionsで実現する\nCI/CDパイプラインの自動化' \
  eyecatch.png
```

改行位置を決めるときの指針:
- 英語の複合名（GitHub Actions、API Gateway、AWS CDK等）を行をまたいで分けない
- カタカナ語の途中で切らない（アプリケーション、サーバーレス等）
- 句読点や助詞（の・を・は等）が行頭に来ないようにする
- 各行の長さがなるべく均等になるようにする
- 1行の目安: フォントサイズ64pxの場合、全角約15文字 / 半角約30文字

### 調整のコツ

- テキストが長い → `--size` を小さく（48〜56あたり）
- テキストが短い → `--size` を大きく（72〜80あたり）
- 自動改行で不自然な位置に切れる → `\n` で改行位置を明示指定する

## 画像の確認

生成後は **必ず** Read ツールで画像ファイルを開いて確認する。

チェックポイント:
- テキストが読みやすいか
- テキストが画像内に収まっているか
- 改行位置が自然か
- ベース画像のロゴや重要な要素と被っていないか

問題があればパラメータを変えて再実行する。ユーザーに「こう変えてみます」と伝えてから再生成すると、フィードバックループがスムーズになる。
