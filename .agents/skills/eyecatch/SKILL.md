# eyecatch: アイキャッチ画像生成

ブログ記事のアイキャッチ画像を生成するスキル。
Pythonスクリプト (`scripts/make_eyecatch.py`) でベース画像にタイトルテキストを重ね、OGP用の画像を作る。

スクリプトは `uv run` で実行するため、Python環境の事前構築は不要。
フォントもWebから自動ダウンロード（初回のみ）されるので、ローカルのフォント環境に依存しない。

## 前提

- `uv` がインストール済みであること
- ベース画像: `.agents/skills/eyecatch/assets/eyecatch.base.png`

## 生成の流れ

1. タイトルテキストを決める（`$ARGUMENTS` にあればそれを使う）
2. 出力先を決める
   - 記事ディレクトリがあれば `blog/<article-id>/images/eyecatch.png`
   - なければプロジェクトルートに `eyecatch_output.png`
3. スクリプトを実行する
4. **生成された画像を Read ツールで確認する**（ここが重要）
5. 問題があればパラメータを調整して再生成する

## スクリプトの実行

プロジェクトルートから実行する:

```bash
uv run .agents/skills/eyecatch/scripts/make_eyecatch.py \
  .agents/skills/eyecatch/assets/eyecatch.base.png \
  "タイトルテキスト" \
  出力先パス \
  [--size フォントサイズ]
```

### オプション

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--size` | 64 | フォントサイズ（px） |
| `--font-url` | Noto Sans CJK JP Bold | 別のWebフォントURL |
| `--font-file` | - | ローカルフォントファイルのパス |

### 調整のコツ

- テキストが長い → `--size` を小さく（48〜56あたり）
- テキストが短い → `--size` を大きく（72〜80あたり）
- テキストの改行位置が不自然 → タイトル文言自体を微調整するか、`--size` で調整

## 画像の確認

生成後は **必ず** Read ツールで画像ファイルを開いて確認する。

チェックポイント:
- テキストが読みやすいか（背景とのコントラスト）
- テキストが画像内に収まっているか
- 改行位置が自然か
- ベース画像のロゴや重要な要素と被っていないか

問題があればパラメータを変えて再実行する。ユーザーに「こう変えてみます」と伝えてから再生成すると、フィードバックループがスムーズになる。

## カスタマイズ

`scripts/make_eyecatch.py` はテンプレートとして提供している。使用者が自由に編集して、プロジェクトに合ったデザインにカスタマイズできる。

よくあるカスタマイズ例:
- テキストの色やフォントの変更
- サブタイトルや著者名の追加
- テキスト位置の調整（中央以外への配置）
- 装飾の追加（背景帯、枠線、グラデーション、アイコン）
- ベース画像の差し替え（`assets/eyecatch.base.png` を入れ替える。スキルディレクトリ内）

スクリプトを編集する場合は、変更内容をユーザーに説明してから行うこと。
