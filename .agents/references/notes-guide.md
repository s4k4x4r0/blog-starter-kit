# NOTES.md 記録ガイド

NOTES.md はプロジェクトの中心的なドキュメントです。コンテキストがクリアされても、このファイルだけで作業を再開できる状態を目指してください。

## 誰が書くか

**メインエージェントだけが NOTES.md を書きます。**

サブエージェントに調査や検証を依頼した場合、サブエージェントは結果をメインエージェントに返すだけです。メインエージェントがその結果を整理して NOTES.md に記録してください。これにより記録の責任が一元化され、何が記録済みかをメインエージェントが常に把握できます。

## いつ書くか

以下のタイミングで即座に NOTES.md を更新してください。次の作業に進む前に記録することで、ユーザーがいつでもコンテキストをクリアできる状態を保ちます。

| タイミング | 記録すること |
|---|---|
| ユーザーとのディスカッションで方針が決まった | 決定事項を該当セクションに記録 |
| 調査で新しい事実がわかった | 調査セクションに追記 |
| 検証で結果が出た（成功・失敗問わず） | ログセクションに追記 |
| 以前の記録に誤りが見つかった | **元の記述を訂正**（ログに経緯を残す） |
| TODO の状態が変わった | TODO セクションを更新 |

## NOTES.md の構造

```markdown
# 検証ノート

## 企画
- 読者ターゲット、記事のゴール、差別化ポイント
- 検証で確認すべきこと

## 技術スタック
- 使用する言語・フレームワーク・ツールとバージョン
- 選定理由

## 調査
### トピック名
- 確定した事実をまとめる（時系列ではなくトピック別）
- 後から訂正があれば、このセクションを直接更新する

## 方針
- 検証の進め方、アーキテクチャ方針

## ブログポイント
- -> ブログ: ログから拾い上げた、記事に書くべきポイントの集約
- -> ブログ: ハマりポイント、意外だった挙動、差別化ネタなど

## TODO
- [ ] 未完了のタスク
- [x] 完了したタスク

## ログ
### YYYY-MM-DD: 見出し
- やったこと、わかったこと
- 失敗したこと -> 原因
- -> ブログ: 記事に書きたいポイント（ブログポイントセクションにも転記）
```

## 各セクションの役割

### 企画・技術スタック・調査・方針（確定情報）

これらのセクションは**現在の正しい状態**を反映し続けてください。

- 新しい事実がわかったら追記する
- 誤りが見つかったら**元の記述を直接修正する**（古い情報を残さない）
- ログ側に「当初〇〇と想定 → 実際は XX だった」と経緯を書くのは OK

例: 調査段階で「Zip デプロイ可能」と記録したが、検証で「コンテナイメージ必須」と判明した場合
- **調査セクション**: 「コンテナイメージ必須」に書き換える
- **ログセクション**: 「Zip デプロイを試みたが失敗。コンテナイメージ方式が必須だった」と記録

### ブログポイント（集約）

ログの中に散在する `-> ブログ:` マーカーの内容を、このセクションに集約してください。記事執筆時にログを全スキャンしなくても、ここだけ見れば書くべきポイントがわかる状態にします。

ログに `-> ブログ:` を書いたら、同じ内容をブログポイントセクションにも追記してください。

### TODO（進捗管理）

現在のタスクをチェックボックス形式で管理します。完了したらチェックを入れてください。再開時に「次に何をすべきか」が一目でわかります。

### ログ（時系列の作業記録）

日付付きの作業記録です。ここには経緯や試行錯誤の過程を残します。

## 何を記録するか

コードを読んだだけではわからない情報を重点的に記録してください:

- なぜその方針・技術を選んだか
- 失敗したアプローチとその理由
- ハマりポイントと解決方法
- 想定と異なった挙動
- ユーザーが「ブログに書きたい」と言ったこと

## 記録しなくてよいもの

- コードを読めばわかること（実装の詳細、設定値など）
- 些末な作業ログ（ファイル作成、依存追加など）

## スクリーンショット画像

スクリーンショットの保存先:

- POC 中 → `poc/images/`
- 記事用 → `blog/<article-id>/images/`

命名は説明的な名前を使い、連番は使わないでください:

- 良い例: `grafana-dashboard-overview.png`, `error-log-detail.png`
- 悪い例: `screenshot1.png`, `image_01.png`

スクリーンショットを撮ったら NOTES.md のログに記録し、`-> ブログ:` マーカーで用途を併記してください。撮影には `screenshot` スキルを使用してください。
